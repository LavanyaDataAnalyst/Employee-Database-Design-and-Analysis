## 📘Project Overview:

> Designed and analyzed an employee database using SQL by creating tables for employees, departments, and locations.
> Performed data operations and queries using joins, aggregate functions, and filtering to analyze workforce data.
> Generated insights on salary distribution, employee demographics, and department performance for decision-making.

## 🗂 Data Source:
* The database consists of three main tables:

**Employees**

Stores employee details such as:
> Employee ID
> Name
> Gender
> Age
> Hire Date
> Designation
> Department ID
> Location ID
> Salary

**Departments**

Contains department-related information:
> Department ID
> Department Name

**Location**

Stores office location details:
> Location ID
> Location Name


## 🛠 Tools & Technologies:

**SQL (MySQL)**

* Joins, subqueries, and aggregate functions

**Relational Database Design**
* Table creation with primary and foreign keys
* Schema design and normalization

**Advanced SQL Concepts**
* GROUP BY and HAVING clauses
* Views and data filtering
* Data analysis using aggregate functions

**Data Analysis**
* Extracting insights from structured data
* Identifying trends in employee salary, age, and performance

## 🔍 Key Insights

* Some departments have fewer employees, indicating possible understaffing.
* Salary varies significantly across departments and roles.
* Certain locations have higher average employee age.
* Analysts have varied salary ranges depending on experience.
* Missing designation data was identified and corrected.

## 💡Recommendations

* Ensure proper data validation during entry to avoid missing values.
* Balance workforce across departments with fewer employees.
* Standardize salary structures for similar roles.
* Monitor hiring trends across locations.
* Improve data consistency for better analysis.
